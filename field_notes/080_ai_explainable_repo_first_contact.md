# AI-Explainable Repo on First Contact

## Observation

A Reddit discussion around AI-readable onboarding produced an important signal.

Some practitioners currently respond with:

- good README is enough
- AGENTS.md / CLAUDE.md is enough
- docs are just docs

This is a useful market signal, not a rejection of the deeper problem.

## Interpretation

The phrase `AI-readable onboarding capsule` may sound like "more docs."

The more immediate value is:

When a user forks or clones an unfamiliar repo, they should be able to ask their own AI:

> Explain this repo to me and tell me the safest first action.

This reframes the idea from future-facing documentation theory into immediate user value.

## Near-term framing

Prefer:

`AI-explainable repo on first contact`

over:

`AI-readable onboarding capsule`

when speaking to general developers.

## Why this matters

As AI-assisted repos, forks, skills, agents, and generated docs increase, users will not read everything manually.

The winning repo may not be the repo with the most documentation.

It may be the repo whose structure lets a user's AI quickly identify:

- where to start
- which files are canonical
- what not to invent
- what not to modify
- what the smallest safe first action is

## V13 interpretation

This is a V13 entry-path problem.

The issue is not documentation volume.

The issue is whether the next user / next AI can re-enter the repo without losing the map, inventing features, or jumping beyond the dependency frontier.
