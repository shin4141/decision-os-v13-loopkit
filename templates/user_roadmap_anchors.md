# User Roadmap Anchors Template

## Purpose

Use this file to define the line that V13 should optimize toward.

V13 does not choose your Aspire.

V13 uses your declared anchors to select better 0.01 repairs.

In one line:

> A 0.01 is only compounding when it moves along a declared line.

## How to Use

Fill in at least two anchors.

One point is only a desire.

Two points create a line.

Three points begin to reveal a trajectory.

## What V13 Should Optimize For

V13 should prefer 0.01 repairs that:

* move toward Anchor 1
* preserve the path to Anchor 2
* do not damage Anchor 3
* preserve Carrier
* avoid speculative feature growth
* avoid public reaction-chasing
* improve the next loop’s starting condition

## What V13 Should Avoid

V13 should avoid:

* local polish that does not support the line
* features that add complexity without moving toward anchors
* public exposure before readiness
* revenue work before proof/adoption signals
* optimization that damages Carrier
* actions that require a Decision Packet but do not ask for one
